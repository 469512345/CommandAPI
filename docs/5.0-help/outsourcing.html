The CommandAPI's Conversion System

Introduction

The CommandAPI has a conversion system which lets server owners convert regular plugin commands into Minecraft-compatible ones.

The CommandAPI has two times when it can run this conversion system:

  • Plugin-load time
  • Plugin-enable time

Depending on when this conversion system is run, changes the scope of where this converted command can be used. If the conversion is run during plugin-load time, then the command can be used anywhere. If the conversion is run during plugin-enable time, then the command cannot be used in datapacks. As a result, plugin-load time is preferred.

To load commands at plugin-load time, the plugin which commands the commands must be loaded before the CommandAPI. In diagram form:

\[ \texttt{Plugin loads} \\ \downarrow \\ \texttt{Plugin loads commands} \\ \downarrow \\ \texttt{CommandAPI loads} \\ \downarrow \\ \texttt{CommandAPI converts plugin commands} \\ \downarrow \\ \texttt{Profit!} \]

If the plugin is loaded before the CommandAPI, then we get the following issue:

\[ \texttt{CommandAPI loads} \\ \downarrow \\ \texttt{CommandAPI converts plugin commands} \\ \downarrow \\ \texttt{CommandAPI fails to find plugin} \\ \texttt{commands because plugin isn't loaded} \\ \downarrow \\ \texttt{CommandAPI cries} \\ \downarrow \\ \texttt{Plugin loads} \\ \downarrow \\ \texttt{Plugin loads commands} \\ \]

To combat this, we currently have to put loadbefore: [CommandAPI] in the plugin.yml file for the plugin that loads the commands. Since this is a pain, I was working on a "deferred system" which will allow server owners to ignore all of this plugin.yml modifications and instead settle for the plugin-enable time conversion. If they wanted to have commands in datapacks, then they could deal with the plugin.yml thing to have plugin-load time conversion.

The problem

During testing, I discovered that even though the plugin loaded after the CommandAPI, the plugin-load time conversion still worked! This whole testing thing that we're trying to do is to confirm whether or not this is just a hiccup with my computer or whether it's actually reliable. If so, command conversion will be much much easier for all server owners!

What you need to do

Prerequisites:

  • A server. Doesn't matter what version you're running
  • The CommandAPI, version 4.3 or 4.3b (who wants to use 4.3a given that it's got bugs in it...) (You can download it from here)
  • Some plugins that register commands. I've tried using EssentialsX, so I'd prefer not using that!

Steps to follow:

  • Shove the CommandAPI onto your server and start it up
  • When the CommandAPI loads for the first time, it creates a folder CommandAPI with a config.yml file.
  • Stop the server
  • Edit the CommandAPI's config.yml file to convert some commands. The instructions for command conversion is outlined here. DO NOT edit the plugin.yml file of any plugins
  • Run the server again
  • Run one of the converted commands using /execute run <the converted command>
